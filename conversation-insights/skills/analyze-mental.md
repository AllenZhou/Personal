# 心智模型分析

分析用户与 AI 协作时的思维方式、问题处理习惯、决策特征，识别潜在盲点。

## 分析维度

### 1. 问题分解习惯

| 类型 | 特征 | 检测方法 |
|------|------|----------|
| 整体思考 | 描述完整目标，不拆分步骤 | 首条消息无编号列表，无"第一步/首先" |
| 拆分思考 | 任务已分解为具体步骤 | 首条消息有编号列表或分步描述 |
| 目标导向 | 强调最终结果 | 消息中包含"最终要"/"目标是" |
| 过程导向 | 关注实现步骤 | 消息中详细描述技术路径 |

**统计**:

- 整体思考占比: X%
- 拆分思考占比: Y%
- 典型案例: session_xxx

### 2. 假设检验方式

| 类型 | 特征 | 检测方法 |
|------|------|----------|
| 先做再验 | 直接执行，遇到问题再调整 | 无 research 前置，直接 new-feature |
| 先验再做 | 先确认可行性，再开始实施 | research → new-feature 序列 |
| 边做边验 | 小步迭代，频繁验证 | 短对话多轮，频繁提问 |

**统计**:

- 先做再验占比: X%
- 先验再做占比: Y%
- 边做边验占比: Z%

### 3. 不确定性应对

| 类型 | 特征 | 检测方法 |
|------|------|----------|
| 容忍模糊 | 信息不完整时仍能推进 | 低追问频率，快速决策 |
| 追求确定 | 需要完整信息才开始 | 高追问频率，要求详细解释 |
| 渐进明确 | 边做边澄清 | 中等追问频率，分阶段推进 |

**检测指标**:

- 平均追问次数/对话
- 要求澄清的频率
- "能详细解释一下"类表达出现次数

### 4. 反馈接收模式

| 类型 | 特征 | 检测方法 |
|------|------|----------|
| 快速采纳 | AI 建议直接接受 | 纠正类型中 user_override 占比 <20% |
| 批判验证 | 对 AI 建议提出质疑 | 纠正类型中 user_override 占比 >40% |
| 选择性采纳 | 部分接受部分调整 | 介于两者之间 |

**统计**:

- 首轮接受率: X%
- 纠正密度: Y 次/对话
- user_override 占比: Z%

### 5. 任务偏好

分析用户擅长和不擅长的任务类型：

| 任务类型 | 对话数 | 平均轮数 | 完成率 | 效率评估 |
|----------|--------|----------|--------|----------|
| debugging | - | - | - | 高效/中等/低效 |
| new-feature | - | - | - | - |
| research | - | - | - | - |
| refactoring | - | - | - | - |
| config | - | - | - | - |

## 执行步骤

### Step 1: 数据收集

读取所有对话，提取以下信息：

- 首条消息内容和结构
- 对话中的追问和澄清请求
- 纠正类型和频率
- task_type 和 outcome 分布

### Step 2: 模式识别

对每个维度进行分类和统计：

1. 分析首条消息结构 → 问题分解习惯
2. 分析对话序列 → 假设检验方式
3. 统计追问频率 → 不确定性应对
4. 分析纠正类型 → 反馈接收模式
5. 按 task_type 统计效率 → 任务偏好

### Step 3: 输出心智模型画像
