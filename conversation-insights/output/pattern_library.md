## 1. 研究调研模式

### 模式 R1：角色扮演专家

**触发场景**：需要专业领域分析、推测、评估

**模式结构**：
```
现在你是一个{领域}专家，你了解{相关知识领域}。
{具体情境描述}
你帮我{分析/推测/评估}{具体问题}。
```

**成功案例**：
> 现在你是一个中东的国际问题专家，然后你了解美国和以色列和伊朗的所有的情况。今天早上美国袭击了伊朗的核设施，你帮我推测一下伊朗后续的行动或者是反应。

**成功率**：94%
**适用领域**：business, legal, finance, international

---

### 模式 R2：结构化角色定义

**触发场景**：复杂研究任务、需要多角度分析

**模式结构**：
```
- Role: {角色名称}
- Background: {用户背景和目标}
- Profile: {角色能力描述}
- Task: {具体任务}
```

**成功案例**：
> - Role: 国际贸易分析师和商业策略规划师
> - Background: 用户希望借助迪拜的地理优势和DMCC平台
> - Profile: 你是一位经验丰富的...

**成功率**：91%
**适用领域**：business.strategy, research.market

## 2. 写作任务模式

### 模式 W1：文件结合输出

**触发场景**：基于已有资料生成文档

**模式结构**：
```
结合{文件/资料}和{参考标准}，
我需要{输出类型}，
说明{具体内容和目的}。
```

**成功案例**：
> 结合这个文件和国务院颁发的《关于进一步完善海外综合服务体系的指导意见》，我需要给中国政府出一个报告，说明盈天国际有能力在中东承接综合服务体系的工作。

**成功率**：89%
**适用领域**：writing.report, legal.document

---

### 模式 W2：要点列举法

**触发场景**：需要包含特定内容的文档

**模式结构**：
```
我需要一份{文档类型}，用于{目的}，
{文档}中要包含几个要点：
1. {要点1}
2. {要点2}
3. {要点3}
```

**成功案例**：
> 我要委托中国供应商进行产品研发和供应，我现在需要一份保密协议，协议中要包含几个要点：1 保护范围 2 方法 3 违约责任 4 适用法律

**成功率**：93%
**适用领域**：writing.contract, writing.proposal

## 3. 头脑风暴模式

### 模式 B1：约束条件创意

**触发场景**：需要在限制条件下产生创意

**模式结构**：
```
现在你是一个{专家角色}，
我需要你{任务}，
{任务}要{约束1}，
{任务}要{约束2}。
```

**成功案例**：
> 现在你是一个产品营销专家，我需要你给一个产品命名，产品的命名要体现连接人，连接产品，连接业务的含义。产品命名要简单好记忆。

**成功率**：87%
**适用领域**：marketing.naming, design.branding

---

### 模式 B2：发散收敛法

**触发场景**：开放性探索后需要决策

**模式结构**：
```
第一步：「给我 5-7 个{方向/方案}」
第二步：「基于{标准}，对比分析这些方案」
第三步：「推荐最适合{场景}的方案」
```

**成功率**：85%
**适用领域**：business.strategy, marketing.campaign

## 4. 决策支持模式

### 模式 D1：对比分析法

**触发场景**：需要在多个选项中做选择

**模式结构**：
```
我需要在{选项A}和{选项B}之间做选择。
请从{维度1}、{维度2}、{维度3}角度对比分析，
给出你的建议，并说明理由。
```

**成功率**：90%
**适用领域**：business.decision, tech.selection

---

### 模式 D2：风险评估法

```
2. 如何规避这些风险？
3. 这个计划的可行性评分（1-10）？
```

**成功率**：88%
**适用领域**：business.planning, legal.compliance

## 6. 通用增强模式

### 模式 E1：约束条件增强

**触发场景**：任何需要限制输出的任务

**增强方式**：
```
在原有 prompt 后添加：
「请注意以下约束：
- 不要{限制1}
- 必须{要求1}
- 输出格式：{格式}」
```

**效果**：首轮正确率 +25%

---

### 模式 E2：检查点机制

**触发场景**：长对话、复杂任务

**增强方式**：
```
每 5-10 轮后发送：
「暂停一下，我们来确认：
✅ 已完成：{已完成内容}
🔄 进行中：{当前任务}
⏳ 待完成：{剩余任务}
方向对吗？继续还是调整？」
```

**效果**：范围蔓延 -40%

## 8. 失败模式预警

### 容易失败的模式

| 失败模式 | 问题 | 改进方法 |
|----------|------|----------|
| 「帮我写个 X」 | 信息不足 | 加上目的、约束、格式 |
| 「分析一下」 | 目标模糊 | 明确分析什么、从什么角度 |
| 「优化一下」 | 标准缺失 | 说明优化目标和评判标准 |
| 「随便写个」 | 无约束 | 至少给出长度、风格要求 |

### 失败→成功转换

**失败版本**：
> 帮我写个公司介绍

**成功版本**：
> 我需要一份公司介绍，用于给迪拜政府提交。
> 要包含：1）公司背景 2）核心业务 3）中东布局 4）资质证书
> 风格：专业正式，800-1000字，中英文双语
