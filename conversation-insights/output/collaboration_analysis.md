
## 3. 协作舞蹈模式

### 典型协作序列

**模式 A：指令-执行型**（最常见，占 65%）
```
你：[指令]
AI：[执行结果]
你：[确认/调整]
AI：[修正]
结束
```

**模式 B：探索-收敛型**（占 20%）
```
你：[开放问题]
AI：[多角度分析]
你：[选择方向]
AI：[深入该方向]
你：[进一步追问]
...
收敛
```

**模式 C：协作-共创型**（占 15%）
```
你：[初始想法]
AI：[扩展建议]
你：[结合反馈调整]
AI：[迭代优化]
共同产出
```

### 协作效率分析

| 模式 | 平均轮次 | 满意度 | 适用场景 |
|------|----------|--------|----------|
| 指令-执行 | 3-5 轮 | 高 | 明确任务 |
| 探索-收敛 | 8-15 轮 | 中高 | 研究分析 |
| 协作-共创 | 10-20 轮 | 中 | 创意任务 |

## 4. 修复机制分析

### 误解发生率

```
每 100 轮对话中：
├── 顺利进行     ████████████████████████████████  95.8%
└── 需要修复     ████░░░░░░░░░░░░░░░░░░░░░░░░░░░░   4.2%
```

### 修复策略分布

| 策略 | 使用频率 | 效果 |
|------|----------|------|
| 补充信息 | 55% | ⭐⭐⭐⭐ 高效 |
| 方向调整 | 30% | ⭐⭐⭐ 中效 |
| 细节修正 | 12% | ⭐⭐⭐⭐ 高效 |
| 否定重试 | 3% | ⭐ 低效 |

### 修复效率评估

**高效修复特征**：
- 说明「不对」的原因
- 补充缺失的信息
- 明确新的方向

**低效修复特征**：
- 只说「不对」
- 重复原始请求
- 期望 AI 猜测意图

### 可预防的修复

```
55% 的修复源于首轮信息不足

典型场景：
├── 未说明约束条件    → AI 给出不符合约束的答案
├── 未说明输出格式    → AI 格式不符合预期
├── 背景信息缺失      → AI 假设错误
└── 任务边界模糊      → AI 范围理解偏差

预防方法：
首轮多花 10 秒，补充约束和格式要求
```
