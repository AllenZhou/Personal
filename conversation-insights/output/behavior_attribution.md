## 3. 认知偏差触发条件

### Scope Creep (58次)

```
触发条件分析：

频率：58 次 / 600 对话 = 9.7%
严重程度：中等

触发场景：
├── 任务定义模糊      ~50%
│   └── 没有明确"完成标志"
├── AI 输出超预期     ~30%
│   └── 看到 AI 能做更多，想顺便加需求
└── 主动探索扩展      ~20%
    └── 好奇心驱动的延伸

高风险信号词：
- "顺便也..."
- "既然都做了..."
- "另外再加个..."
- "还有一个小问题..."
```

```
触发条件分析：

频率：12 次 / 600 对话 = 2%
严重程度：轻微

触发场景：
├── 重要文档输出      ~60%
│   └── 对外文档、正式报告
├── 方案选择决策      ~25%
│   └── 反复比较不同选项
└── 代码优化          ~15%
    └── 追求更优实现

高风险信号词：
- "再优化一下"
- "还不够好"
- "能不能更..."
```

## 8. 预防性建议

### 基于归因的预防策略

| 归因类型 | 预防动作 | 预期效果 |
|----------|----------|----------|
| missing_context | 首轮使用 CRISP 框架 | 纠正率 -40% |
| ai_misunderstand | 提供具体示例 | 纠正率 -25% |
| scope_change | 写明"完成标志" | 范围蔓延 -50% |
| user_changed_mind | 开始前 2 分钟思考 | 方向变更 -30% |

### CRISP 框架

```
C - Context（背景）：项目、环境、约束
R - Role（角色）：希望 AI 扮演的角色
I - Intent（意图）：要实现的具体目标
S - Scope（范围）：做什么、不做什么
P - Product（产出）：期望的输出格式
```

### 检查清单

```
开始对话前（30 秒检查）：
□ 背景信息完整了吗？
□ 目标表述清晰了吗？
□ 有没有明确边界？
□ 完成标志是什么？
```
