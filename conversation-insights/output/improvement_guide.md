## P0：提升 Prompt 质量

**当前状态**：平均 45.1/100，高质量样本（70+分）仅占 15%

### 行动 1：使用 CTAO 框架

每次提问前，花 10 秒检查这 4 个要素：

```
C - Context（背景）: 我在做什么项目？目前进展到哪一步？
T - Task（任务）: 具体要 AI 做什么？
A - Audience（受众）: 输出给谁看？什么场景用？
O - Output（输出）: 要什么格式？长度？语言？
```

**❌ 低分示例**（score: 35）
> 帮我写个文案

**✅ 改进后**（score: 80）
> 背景：我在推广阿联酋公司注册服务，目标客户是中国出海企业。
> 任务：写一篇微信公众号文案，介绍 DMCC 自贸区的税务优势。
> 要求：800-1000 字，专业但不枯燥，包含具体数据。

### 行动 2：复用高分模板

你的高分 Prompt 有共同结构，直接套用：

**研究类任务**：
```
现在你是一个{领域}专家，你了解{相关知识}。
{当前情境}
你帮我{分析/评估/推测}{具体问题}。
```

**文档类任务**：
```
我需要一份{文档类型}，用于{目的}，
要包含以下要点：
1. {要点1}
2. {要点2}
3. {要点3}
输出格式：{Markdown/Word/具体要求}
```

### 行动 3：保存常用 Prompt

在 Claude Code 中创建 Prompt 快捷方式：
```bash
# ~/.claude/prompts/research.md
# ~/.claude/prompts/writing.md
# ~/.claude/prompts/brainstorm.md
```

## P1：提高问题分解率

**当前状态**：仅 12.2% 的对话使用结构化多步骤

### 行动 1：复杂任务强制拆分

**触发条件**：任务描述超过 50 字 或 涉及多个步骤

**执行步骤**：
```
在发送 Prompt 前，先写：
"这个任务分为以下步骤：
1. ...
2. ...
3. ...
我们先从第 1 步开始。"
```

**❌ 未分解**
> 帮我分析迪拜公司注册流程、成本、税务优惠，然后写个方案给客户

**✅ 分解后**
> 这个任务分为 3 步：
> 1. 先整理迪拜公司注册的基本流程
> 2. 对比不同自贸区的成本和税务优惠
> 3. 基于以上内容写客户方案
>
> 我们先从第 1 步开始：迪拜公司注册的基本流程是什么？

### 行动 2：使用"首先-然后"关键词

在 Prompt 中明确使用序列词：

```
首先，帮我...
然后，基于上面的结果...
最后，汇总成...
```

### 行动 3：设置检查点

超过 5 轮对话后，主动问：

> "我们目前完成了什么？接下来还有什么要做？"

## P2：控制范围蔓延

**当前状态**：27 次范围蔓延，占 4.5%

### 行动 1：开始时声明边界

**模板**：
```
【这次对话的目标】
只做：{具体任务}
不做：{明确排除的内容}
完成标志：{怎么算完成}
```

**示例**：
```
【这次对话的目标】
只做：修复登录页面的样式问题
不做：添加新功能、重构代码
完成标志：登录页面在移动端正常显示
```

### 行动 2：识别范围蔓延信号

当你发现自己说以下话时，警惕范围蔓延：

| 信号 | 示例 | 应对 |
|------|------|------|
| "顺便..." | "顺便把那个也改了" | 记下来，开新对话 |
| "既然都做了..." | "既然都做了不如也加上 X" | 评估是否必要 |
| "再加一个..." | "再加一个小功能" | 确认是否在原始范围内 |

### 行动 3：使用"停车场"机制

发现新想法时：
```
"这个先记到停车场，等当前任务完成再处理：
- 停车场：{新想法}

现在继续：{当前任务}"
```

## P3：管理长会话

**当前状态**：长会话(>20轮)纠正密度是短会话的 1.75 倍

### 行动 1：设置轮次预警

| 轮次 | 状态 | 建议行动 |
|------|------|----------|
| 1-10 | 🟢 正常 | 继续 |
| 11-20 | 🟡 警惕 | 检查进度，考虑总结 |
| 20+ | 🔴 疲劳 | 总结当前进度，开新对话 |

### 行动 2：主动总结

超过 15 轮时，发送：
```
"暂停一下，我们来总结：
✅ 已完成：...
🔄 进行中：...
⏳ 待完成：...

继续还是开新对话？"
```

### 行动 3：分会话策略

复杂项目拆分为多个独立会话：

```
会话 1：需求分析
会话 2：技术方案
会话 3：代码实现
会话 4：测试和优化
```

## 每周检查清单

**周一开始前**（5 分钟）：
- [ ] 查看本周主要任务
- [ ] 为复杂任务预先写好分步 Prompt

**每次对话开始**（10 秒）：
- [ ] 用 CTAO 框架检查 Prompt
- [ ] 如果任务复杂，先拆分步骤

**对话进行中**：
- [ ] 第 10 轮：检查是否偏离目标
- [ ] 发现新想法：用停车场机制
- [ ] 第 20 轮：强制总结或开新对话

**周五结束**（10 分钟）：
- [ ] 运行 `python scripts/enrich_helper.py batch-digest --limit 20`
- [ ] 检查本周对话质量趋势
