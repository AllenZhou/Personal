## 2. 任务类型风险预测

### 高风险任务预警

| 任务特征 | 风险等级 | 预测问题 | 预防措施 |
|----------|----------|----------|----------|
| **new-feature + 长对话** | 🔴 高 | 范围蔓延 | 写明完成标志 |
| **debugging + 信息不全** | 🔴 高 | 多轮纠正 | 首轮提供环境信息 |
| **design + 无约束** | 🟡 中 | 方向反复 | 先确认风格偏好 |
| **research + 宽泛主题** | 🟡 中 | 发散失焦 | 分解为具体问题 |

### 任务开始时的风险检测

```
当你开始以下类型的任务时，系统会预警：

if 任务类型 == "new-feature":
    预警：范围蔓延风险 +40%
    建议：明确写出"完成标志"和"不做的事项"

if 任务类型 == "debugging" and 缺少环境信息:
    预警：纠正率 +60%
    建议：提供：OS、版本、错误日志、已尝试方法

if Prompt 长度 < 20 字:
    预警：首轮正确率 -30%
    建议：补充背景和约束信息
```

## 3. 认知陷阱预警

### 你的高频陷阱

| 陷阱 | 历史频率 | 触发预测 | 实时预警信号 |
|------|----------|----------|--------------|
| **Scope Creep** | 9.7% | 模糊任务定义 | "顺便也..." |
| **Perfectionism** | 2.0% | 重要文档 | "再优化一下" |
| **Information Overload** | 估 5% | 长对话 | 消息变短 |

### 实时预警机制

```
对话进行中，以下信号触发预警：

范围蔓延信号：
├── "顺便也..." → 预警：停下来确认是否在目标范围内
├── "另外再加个..." → 预警：先完成当前任务
└── 对话超过 15 轮 → 预警：检查是否偏离原目标

疲劳信号：
├── 连续 3 条消息 < 10 字 → 预警：可能在赶进度
├── 跳过验证输出 → 预警：可能在妥协质量
└── "算了就这样吧" → 预警：可能是疲劳妥协

完美主义信号：
├── 同一内容修改 > 3 次 → 预警：是否在追求不必要的完美
└── "能不能更好一点" → 预警：明确"更好"的标准
```

## 7. 预防性干预建议

### 任务开始前（30 秒检查）

```
自动检查清单：

□ 任务类型是高风险类型吗？
  → new-feature / debugging 需要更详细的 Prompt

□ Prompt 长度足够吗？
  → < 20 字：补充背景和约束
  → < 50 字：考虑增加细节

□ 有明确的完成标志吗？
  → 复杂任务：写明"做完什么算完成"

□ 这是你的强项领域吗？
  → 弱项领域：降低期望，多验证
```

### 任务进行中（实时监控）

```
触发预警的信号：

轮次达到 10 轮：
→ 暂停检查：还在原目标上吗？

连续短消息（< 10 字）：
→ 暂停检查：是疲劳还是在赶进度？

发现自己说"顺便"：
→ 暂停：先完成当前任务

同一内容修改 3 次：
→ 暂停：是否在追求不必要的完美？
```

### 任务结束后（复盘检查）

```
快速复盘（1 分钟）：

□ 完成了原始目标吗？
□ 有范围蔓延吗？
□ 哪里可以做得更好？
□ 有可复用的模式吗？
```
