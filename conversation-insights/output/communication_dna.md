### 输出结构期望

```
结构化（表格/列表）   ████████████████████████████████  65%
半结构化（分段落）    ████████████████░░░░░░░░░░░░░░░░  30%
非结构化（纯文字）    ████░░░░░░░░░░░░░░░░░░░░░░░░░░░░   5%
```

### 结构不对称性

**核心发现**：你习惯用非结构化方式提问，但期望得到结构化回答。

```
输入：「帮我分析一下这个」     → 非结构化
期望：表格对比、步骤列表      → 结构化
```

**建议**：
- 输入也用结构化，可以得到更精准的结构化输出
- 明确说「用表格对比」「列出步骤」

## 4. 表达风格分析

### 指令风格

| 风格 | 频率 | 示例 |
|------|------|------|
| **直接指令** | 72% | 「帮我写...」「分析...」 |
| **请求式** | 18% | 「能不能...」「可以...吗」 |
| **协商式** | 10% | 「你觉得...」「建议...」 |

### 常用句式模式

```
「帮我 + [动作] + [对象]」             → 最常用
「我需要 + [输出] + [用途]」           → 较常用
「现在你是 + [角色]，[任务]」          → 高质量模式
「结合 + [文件/资料]，[任务]」         → 高质量模式
```

### 隐含 vs 显式信息

| 类型 | 比例 | 影响 |
|------|------|------|
| **显式信息** | 60% | 直接说明的需求 |
| **隐含假设** | 40% | 期望 AI 自动理解 |

**常见隐含假设**：
- 输出语言（默认中文）
- 输出长度（默认适中）
- 专业程度（默认匹配你的水平）
